# 提案書スキル

## ミッション

「読んだだけで決断できる」提案書を作る。
読み手の疑問を先回りして潰し、「YES」を言いやすくする。

---

## 実行手順

### Step 1: 事前確認

`templates/user-profile.md` を読んでユーザーの事業・強み・価格帯を把握する。

以下をユーザーに確認する（AskUserQuestion で1問ずつ）:

1. 誰に向けた提案書ですか？（相手の事業・状況）
2. 提案する内容・サービスを教えてください
3. 価格・納期はありますか？
4. 相手が今一番困っていることは何ですか？
5. 競合と比較して、あなたの強みはどこですか？

### Step 2: 構成を提示して確認

```
【提案書構成案】
1. 表紙（タイトル・日付・宛名）
2. 課題の整理（相手の状況・困っていること）
3. 解決策の提示（あなたのサービスで何が解決するか）
4. 実績・根拠（なぜあなたなのか）
5. 提供内容・スコープ（何をどこまでやるか）
6. 料金・納期
7. 次のステップ（いつまでにどうすれば進むか）

この構成で進めてよいですか？
```

### Step 3: 提案書を執筆する

**書き方の原則:**
- 相手の言葉で書く（こちらの専門用語は使わない）
- 「あなたの課題は〇〇です」から始めて共感を示す
- ベネフィット（得られる結果）を先に書いて、機能の説明は後
- 数字で根拠を示す（「集客が増える」ではなく「月3件の新規が見込める」）
- 各セクションに大見出し・サブ見出しを入れる

**禁止パターン:**
- 自社の自慢から始める（相手の課題から始める）
- 「ご検討ください」で終わる（次のアクションを明示する）
- 料金だけ書いて根拠がない（なぜその価格かを説明する）

### Step 4: 見積書（料金明細）が必要な場合

```
【見積書】
項目名 | 内容 | 金額
〇〇 | 〇〇 | ¥〇〇
〇〇 | 〇〇 | ¥〇〇
合計 | | ¥〇〇
有効期限: 〇〇
```

## 納品チェックリスト（全項目○でなければ出力しない）

- [ ] 相手の課題から始まっている（自社自慢から始まっていない）
- [ ] 各セクションに大見出し・サブ見出しがある
- [ ] ベネフィット（結果）が先に書かれている
- [ ] 数字による根拠が最低1箇所ある
- [ ] 料金・納期が明記されている
- [ ] 次のアクションが明示されている（「ご検討ください」で終わっていない）
- [ ] 相手の言葉・業界用語で書かれている
- [ ] 省略なし: 全セクション書き切り
- [ ] 絵文字: なし
