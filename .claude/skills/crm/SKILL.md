# 顧客管理スキル

## ミッション

「〇〇さんの状況教えて」「〇〇さんに連絡した」だけで顧客情報を記録・確認できるようにする。

---

## 「〇〇さんの情報を登録して」「〇〇さんを追加して」

`memory/顧客/〇〇.md` を新規作成する。

記録フォーマット:
```yaml
name: 〇〇
company: 〇〇（任意）
email: 〇〇（任意）
last_contact: YYYY-MM-DD
next_action: 〇〇
memo:
  - YYYY-MM-DD: 〇〇
```

AskUserQuestion で不足情報を1問ずつ確認する。

---

## 「〇〇さんに連絡した」「〇〇さんとやり取りした」

`memory/顧客/〇〇.md` の `last_contact` を今日の日付に更新する。
`memo` に今日の日付と内容を追記する。

---

## 「〇〇さんの状況教えて」「〇〇さんどうなってる？」

`memory/顧客/〇〇.md` を読んで以下を表示する:

```
【〇〇さん】
最終連絡: 〇月〇日（〇日前）
次のアクション: 〇〇
メモ:
- 〇月〇日: 〇〇
```

---

## 「顧客一覧」「お客さん一覧」

`memory/顧客/` 配下の全ファイルをGlobで取得して一覧表示する:

```
【顧客一覧】N名

フォローが必要（30日以上連絡なし）:
- 〇〇さん（最終連絡: 〇月〇日）

最近連絡あり:
- 〇〇さん（最終連絡: 〇月〇日）
```

---

## 「〇〇さんの次のアクションを〇〇にして」

`memory/顧客/〇〇.md` の `next_action` を更新する。更新前に必ずReadで確認してから編集する。

---

## 禁止事項

- 実際にファイルを読まずに「〇〇さんの情報です」と言う
- 絵文字の使用

---

## 品質チェックリスト

- [ ] memory/顧客/ を実際に読んでから報告している
- [ ] last_contact を今日の日付で更新している（ハードコードしていない）
- [ ] 30日以上連絡なしの顧客をフォローアップ欄に表示している
- [ ] 絵文字: なし
