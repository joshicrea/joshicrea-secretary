# skill-creator スキル

## ミッション

あなたの仕事に合わせたスキルを自動生成して `.claude/skills/` に追加する。
使えば使うほど、あなた専用のAI秘書に育っていく。

---

## 実行手順

### Step 1: 要件のヒアリング

`templates/user-profile.md` を読んでユーザーの事業・ツールを把握する。

AskUserQuestion で1問ずつ確認する。

1. どんなスキルを作りますか？（例: 「議事録スキル」「見積書スキル」「週報スキル」）
2. このスキルを使うとき、毎回確認すべき情報は何ですか？
3. 出力の形式に決まりはありますか？（例: 箇条書き・表形式・段落）
4. これまでに「こう書いたらうまくいった」パターンがあれば教えてください
5. 絶対に入れてはいけないこと・やってはいけないことはありますか？

### Step 2: 設計を提示して確認

```
【スキル設計案】
スキル名: 〇〇
目的: 〇〇
事前確認事項: 〇〇・〇〇
実行手順: Step 1〜N
出力フォーマット: 〇〇
禁止事項: 〇〇
チェックリスト: N項目

この設計で進めてよいですか？
```

AskUserQuestion: [OK] [修正したい]

### Step 3: SKILL.md を生成する

以下の要素を全て含む SKILL.md を生成する（省略禁止）。

**必須要素（全て含める）:**

1. **frontmatter**: `name`・`description` を記載する
2. **ミッション**: このスキルが何を達成するかを1行で
3. **実行手順**:
   - Step 1: `templates/user-profile.md` を読む
   - Step 2: AskUserQuestion で事前確認（1問ずつ）
   - Step 3以降: 具体的な作業手順（ステップバイステップ）
4. **Claude Codeの能力を引き出すルール**（以下を毎回含める）:
   - 独立した作業は並列で実行する
   - ファイルを編集する前に必ず Read で確認する
   - 判断が分かれる場面は AskUserQuestion で確認する
   - 推測で報告しない（実物確認後に報告する）
5. **出力フォーマット**: 見出し構成・文体ルール・文字数基準
6. **禁止パターン**: やってはいけないこと（箇条書き）
7. **納品チェックリスト**: 出力前に全項目確認（チェックボックス形式）
   - 「全項目○でなければ出力しない」を必ず明記する
   - 「省略なし: 全文書き切り」を必ず含める
   - 「絵文字: なし」を必ず含める

### Step 4: ファイルに保存する

`.claude/skills/{スキル名}/SKILL.md` として保存する。

保存後、学習ログに記録する:
`memory/学習ログ/YYYY-MM-DD.md` に「{スキル名}スキルを追加した」と追記する。

「{スキル名}スキルを追加しました（.claude/skills/{スキル名}/SKILL.md）。
「{スキル名}を〇〇して」と言えばいつでも使えます。」

---

## 品質チェックリスト（全項目○でなければ出力しない）

- [ ] frontmatter: name・descriptionあり
- [ ] Step 1: user-profile.md を読む指示あり
- [ ] 事前確認: AskUserQuestion で1問ずつ聞く設計になっている
- [ ] Claude Codeの能力を引き出すルールが含まれている
- [ ] 出力フォーマット: 具体的な形式・文字数基準あり
- [ ] 禁止パターン: 最低3項目
- [ ] チェックリスト: 最低5項目、「省略なし」「絵文字なし」を含む
- [ ] 「全項目○でなければ出力しない」の記載あり
- [ ] 省略なし: 全文書き切り
