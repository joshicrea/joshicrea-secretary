# skill-creator スキル

## ミッション

「投稿スキルと同レベルの品質」で、ユーザーが必要とする新しいスキルを自動生成して
`.claude/skills/` に追加する。

---

## 実行手順

### Step 1: 要件のヒアリング

AskUserQuestion で1問ずつ確認する。

1. どんなスキルを作りますか？（例: 「求人票作成スキル」「セミナー告知文スキル」）
2. このスキルを使うとき、毎回確認すべき情報は何ですか？（例: ターゲット・目的・文字数）
3. 出力の形式・フォーマットに決まりはありますか？（例: 箇条書き・表形式・段落形式）
4. これまでに「こう書いたらうまくいった」というパターンがあれば教えてください
5. 絶対に入れてはいけないこと・やってはいけないことはありますか？

### Step 2: 生成するスキルの設計を提示

```
【スキル設計案】
スキル名: 〇〇
目的: 〇〇
事前確認事項: 〇〇・〇〇・〇〇
実行手順: Step 1〜N
出力フォーマット: 〇〇
禁止事項: 〇〇
チェックリスト: 〇〇・〇〇・〇〇

この設計で進めてよいですか？
```

AskUserQuestion: [OK] [修正したい]

### Step 3: SKILL.md を生成する

確認が取れたら以下のフォーマットで SKILL.md を生成する。

**生成するスキルに必ず含める要素（全て必須）:**

1. **frontmatter**: `name`・`description` を記載する
2. **ミッション**: このスキルが何を達成するかを1行で
3. **実行手順**:
   - Step 1: 事前確認（templates/user-profile.md を読む + ユーザーへの確認事項）
   - Step 2以降: 具体的な作業手順（ステップバイステップ）
4. **出力フォーマット**: 見出し構成・文体ルール・文字数基準
5. **禁止パターン**: やってはいけないこと（箇条書き）
6. **納品チェックリスト**: 出力前に全項目確認する（チェックボックス形式）

**チェックリストのルール:**
- 「全項目○でなければ出力しない」を必ず明記する
- 項目は最低5個、最大15個
- 「省略なし: 全文書き切り」を必ず含める
- 「絵文字: なし」を必ず含める

### Step 4: ファイルに保存する

`.claude/skills/{スキル名}/SKILL.md` として保存する。

保存後:
「{スキル名}スキルを追加しました。
「{スキル名}を書いて」と言えばいつでも使えます。」

---

## 品質チェックリスト

生成したスキルが以下を満たしているか確認する。

- [ ] frontmatter: name・descriptionあり
- [ ] Step 1: user-profile.md を読む指示あり
- [ ] 事前確認: ユーザーへの確認事項が明記されている
- [ ] 出力フォーマット: 具体的な形式・文字数基準あり
- [ ] 禁止パターン: 最低3項目
- [ ] チェックリスト: 最低5項目、「省略なし」「絵文字なし」を含む
- [ ] 「全項目○でなければ出力しない」の記載あり
