# 投稿スキル

## ミッション

読んだ人が「もっとこの人の話を聞きたい」と感じる記事を書く。
SEOと読者体験の両立を目指す。

---

## 実行手順

### Step 1: 事前確認

`templates/user-profile.md` を読んでユーザーの文体・ターゲット・ブログ媒体を把握する。

以下をユーザーに確認する（AskUserQuestion で1問ずつ）:

1. 記事のテーマを教えてください
2. この記事を読んだ人に何をしてほしいですか？（LINEに来てほしい・問い合わせ・セミナー申し込み等）
3. 文字数の希望はありますか？（指定なければ2000-3000字で作成します）

### Step 2: 構成設計

書き始める前に構成を提示して確認を取る。

```
【構成案】
テーマ: 〇〇
ターゲット: 〇〇が悩んでいる〇〇さん

H1（タイトル）: 〇〇
H2（1）: 〇〇
H2（2）: 〇〇
H2（3）: 〇〇
まとめ: 〇〇
CTA: 〇〇

この構成で進めてよいですか？
```

AskUserQuestion: [OK] [修正したい]

### Step 3: 記事執筆

確認が取れたら全文を書く。途中省略は禁止。

**媒体別の書き方:**

**Ameblo:**
- スマホで読みやすい短い段落（3-4行が目安）
- 太字・下線で重要箇所を強調する
- 絵文字は使わない
- 改行を多用する

**note:**
- Markdown形式（見出し・引用・区切り線を使う）
- 少し長めの段落でも読みやすい
- 有料記事の場合は無料部分で「続きが気になる」状態を作る

**WordPress:**
- H2・H3の階層を使って構成を明確にする
- SEOキーワードを自然に入れる（詰め込まない）
- リード文（冒頭200字）で検索意図を解決できると伝える

**共通ルール:**
- 一人称の体験・ストーリーを入れる
- 「あなたは〇〇ですか？」の問いかけを使う
- 結論を先に書いてから理由を説明する
- 専門用語は使わないか使う場合は必ず説明する

### Step 4: CTA（記事末尾）

記事の最後に必ず行動を促す文章を入れる。

```
〇〇について詳しく知りたい方は、
[ユーザーのLINEリンク / 問い合わせページ / セミナーページ]
からどうぞ。
```

### Step 5: 自己検証

出力前に全項目を確認する。

---

## 禁止パターン

- 途中省略（「〜以下略」「残りは省略します」等）
- CTAのない記事
- 専門用語の説明なし多用
- コピーライティング丸出しの押しつけ感
- 絵文字の使用

---

## 納品チェックリスト（全項目○でなければ出力しない）

- [ ] 構成: H1・H2・H3の階層が整っている
- [ ] 文字数: 指定文字数の±10%以内
- [ ] リード文: 冒頭3文で「この記事を読む理由」が伝わる
- [ ] 体験・ストーリー: 最低1箇所
- [ ] 問いかけ: 最低1箇所
- [ ] CTA: 記事末尾にあり、具体的な行動先が明記されている
- [ ] 省略なし: 全文書き切り
- [ ] 媒体別フォーマット: 指定媒体の形式に従っている
- [ ] 絵文字: なし
