# ステップメールスキル

## ミッション

登録から成約まで、読者を自然に導くシナリオを設計する。
「売り込み感」なく、信頼を積み上げながら行動を促す。

---

## 実行手順

### Step 1: 事前確認

`templates/user-profile.md` を読んでユーザーの事業・商品・ターゲットを把握する。

以下をユーザーに確認する（AskUserQuestion で1問ずつ）:

1. ステップメールの最終ゴールは何ですか？（例: 個別相談の申し込み、商品購入、セミナー参加）
2. 何通のシナリオを作りますか？（目安: 7通・14通・21通）
3. 登録してくれる人はどんな人ですか？（悩み・状況・きっかけ）
4. 商品・サービスの価格帯を教えてください

### Step 2: シナリオ全体設計を提示

```
【ステップメールシナリオ設計】

最終ゴール: 〇〇
通数: 〇通（配信間隔: 〇日ごと）

各通の役割:
1通目（即日）: 自己紹介 + 価値提供 → 信頼の第一歩
2通目（1日後）: 問題提起・共感 → 痛みの言語化
3通目（3日後）: 解決策の提示 → 教育
4通目（5日後）: 実績・事例 → 証拠
5通目（7日後）: よくある疑問への回答 → 反論つぶし
6通目（9日後）: 限定オファー予告 → 期待感
7通目（11日後）: 本オファー → 行動喚起

この設計で進めてよいですか？
```

AskUserQuestion: [OK] [通数・間隔を変えたい] [構成を変えたい]

### Step 3: 各通を執筆する

確認が取れたら全通を執筆する。各通に以下を含める。

**件名（各通）:**
- 15文字以内
- 数字 or 好奇心フック
- 前の通との連続性を感じさせる

**本文構成（各通）:**
1. 冒頭3行: 読む理由の提示
2. 本題: その回のメインメッセージ
3. CTA: その回の行動（返信して・チェックして・読んで 等）
4. PS: 本文を読み飛ばした人にCTAを届ける

**各通の文字数:** 500-1000字

### Step 4: 自己検証

全通を出力前に確認する。

---

## 禁止パターン

- 1通目からいきなり売り込む
- CTAのない通
- 各通のテーマがバラバラで一貫性がない
- 最終通まで信頼構築だけで行動喚起がない
- 5行以上の段落
- 絵文字の使用

---

## 納品チェックリスト（全項目○でなければ出力しない）

- [ ] シナリオ全体にゴールへの流れがある
- [ ] 各通に件名（3案）・本文・CTA・PSがある
- [ ] 件名: 各通15文字以内、好奇心フックあり
- [ ] 各通500字以上
- [ ] 段落: 全段落5行以内
- [ ] 1通目は信頼構築から始まっている（売り込みなし）
- [ ] 最終通はオファー・行動喚起がある
- [ ] 省略なし: 全通書き切り
- [ ] 絵文字: なし
