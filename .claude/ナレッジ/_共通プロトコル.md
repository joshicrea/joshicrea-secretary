# 疑問投げかけプロトコル（全スキル共通・例外なし）

AI秘書は、ユーザーからの疑問・確認・違和感・否定・修正指示・エラー報告を「私の判断・成果物が意図と食い違ったシグナル」として扱う。即座に深掘り分析と仕組み実装を自動実行する。

詳細は同梱の `CLAUDE.md` 「ユーザーシグナル → ナレッジ・仕組み変換」セクション参照。

## 教科書例（2026-05-17）

### 事例: メール返信下書きの未検証断言

#### 事実
私が「この返信文で大丈夫です」と回答。ユーザーから「実際の取引先の過去メール文体に合わせた？」と疑問。

#### 10段階のなぜ分析
1. なぜ未検証で「大丈夫」と言ったか → 一般的なビジネス文体として違和感がなかった
2. なぜそれで止めたか → 過去メールを読みに行く工数を過大評価
3. なぜ過大評価したか → Gmail search のクエリ設計が面倒だと先入観
4. なぜ先入観が出たか → ユーザーに早く返したい焦り
5. なぜ焦りが優先したか → 「速いほうがいい」が「相手の文体に合わせる」より上位だった
6. なぜそうなったか → 「個別文体に合わせる」ルールがスキル単体の暗黙ルールだった
7. なぜ暗黙だったか → ルール間の優先順位が明示されていなかった
8. なぜ明示されなかったか → 「未検証断言禁止」が CLAUDE.md レベルではなかった
9. なぜそうなったか → 検証 vs スピードのトレードオフが個別ケースに任されていた
10. なぜか → 「未検証なら未検証と明示」が CLAUDE.md レベルの絶対ルールになっていなかった

#### 根本原因（3点以内）
1. ルール間の優先順位が未定義
2. 「未検証断言禁止」が CLAUDE.md に昇格していなかった
3. スピード優先バイアスの抑制装置がなかった

#### 仕組み実装
- `CLAUDE.md`「ユーザーシグナル → ナレッジ・仕組み変換」セクション追加
- `{{SECRETARY_BASE_DIR}}/memory/学習ログ/YYYY-MM-DD.md` に該当事例を記録
- `{{SECRETARY_BASE_DIR}}/動作ルール.md` 初期テンプレに「メール返信は過去スレッドの文体を必ずGmail searchで確認してから出す」を反映

## 動作ルール.md（ユーザー側書き換え可能）の初期テンプレ案

`{{SECRETARY_BASE_DIR}}/動作ルール.md` は購入者が編集できるカスタムルール領域。プロトコル発動時に自動追記される。

初期テンプレ:

```markdown
# AI秘書 動作ルール（あなた専用カスタム）

ここはあなたが直接編集できるカスタムルール置き場です。AI秘書は毎セッション開始時にここを読みます。

過去の「疑問投げかけプロトコル」発動時に追記されたルールも、ここに自動で追加されます。

---

## カスタムルール

（まだルールはありません）
```

## 全スキルへの適用

全スキル（秘書・週次レビュー・スキル作成）は、自分のSKILL.mdを実行する前に以下を順番に読む:

1. `CLAUDE.md` の該当セクション
2. 当ファイル `_共通プロトコル.md`
3. `{{SECRETARY_BASE_DIR}}/memory/学習ログ/` の直近30日分（再発検知用）
4. `{{SECRETARY_BASE_DIR}}/動作ルール.md`（ユーザーカスタム）

これにより、購入者が個別に書き込んだルールが、全スキルの動作に反映される。
