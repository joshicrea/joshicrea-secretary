## 禁止パターン（5項目以上・それぞれ「なぜNG・どう直す」付き）

1. **[禁止パターン1]** → NGな理由: [〇〇だから]。代わりに: [〇〇する]
2. **[禁止パターン2]** → NGな理由: [〇〇だから]。代わりに: [〇〇する]
3. **推測で報告する** → NGな理由: 実際と違う情報を渡すことになる。代わりに: 実物を確認してから報告する。
4. **ユーザーに補完を求める** → NGな理由: ユーザーの手間を増やす。代わりに: 自分で取得・確認して完結した形で渡す。
5. **「〇〇と思われます」「〇〇のはずです」で報告する** → 確認していない情報を事実のように伝えている。代わりに: 確認してから報告するか「確認します」と伝える。
